# Farleyworld

A small MMO-style "lounge" engine built on Rails 8 + ActionCable + Three.js,
rendered in the style of PS1-era FF7 / Resident Evil: fixed cameras,
pre-rendered backgrounds, chunky low-poly characters walking on a 3D mesh.

## Running it

```sh
bin/rails db:prepare db:seed
bin/rails server
```

Open http://localhost:3000, enter a name, walk around. Open a second browser
(or an incognito window) with another name to see multiplayer sync.

- **WASD / arrow keys** — walk (camera-relative)
- **SPACE** — interact with the nearest object or player / dismiss dialog
- Walk into a doorway to change rooms (Lounge ⇄ Courtyard)

### Dev shortcuts

- `http://localhost:3000/?as=Name` — instant login, skips the title screen
- `…&room=courtyard` — spawn in a specific room
- `…&preview=1` — offline preview: no networking, fake players staged, one
  walks in a circle. Useful for eyeballing rendering and room design.

## How it works

### The PS1 trick (client)

`app/javascript/game/world.js` renders the static room geometry **once** into
a color + depth render target (`prerenderBackground`). Every frame afterwards:

1. A fullscreen quad draws that image and writes the *stored per-pixel depth*
   into the depth buffer (`gl_FragDepth`).
2. Only the animated characters are rendered on top, depth-tested against the
   pre-rendered scene — so players are correctly occluded when they walk
   behind pillars, exactly like FF7's pre-rendered backgrounds.

The whole frame is rendered at 1/3 resolution and upscaled with
`image-rendering: pixelated` for the period-correct look (`RENDER_SCALE` in
`game/main.js`).

### Walkmesh

Rooms define a triangle ground mesh (`walkmesh`). Characters exist as 2D
(x, z) coordinates; their height is barycentrically interpolated from the
triangle under them (`game/walkmesh.js`), which makes ramps and platforms
work for free. Movement off the mesh slides along walls. The **server runs
the same point-in-mesh test** (`Room#contains?`) and rejects illegal moves.

### State sync (server)

`RoomChannel` (`app/channels/room_channel.rb`) streams per-room:

| message         | meaning                                          |
|-----------------|--------------------------------------------------|
| `roster`        | sent to you on subscribe: your state + everyone else's |
| `player_joined` / `player_left` | presence changes                 |
| `player_moved`  | validated position broadcast (~10 Hz per client) |
| `interaction`   | someone pressed SPACE near something             |
| `room_change`   | you walked into an exit; resubscribe to continue |

Players are identified by a signed `player_id` cookie (name-only login, no
passwords — it's a lounge).

### Rooms are data

Everything about a room lives in `Room#data` (JSON), seeded from
`db/seeds.rb`: spawn point, fixed camera, lights, the walkmesh (generated by
`grid_walkmesh` with a height function and blocked-cell predicate), visual
props (boxes/cylinders/cones/spheres the client assembles), interactables
with flavor text, and exits. Adding a room = adding a seed entry; no code
changes needed on the client or server.
